# OPB_FINAL_MT Trading System – Complete User Guide

> Last updated: 12 May 2026 | Version: 2.45

---

## 1. System Overview

**Architecture:** Clean Architecture (Ports & Adapters) with DI container
**Modules:** 100+ core modules, 1500+ tests
**Brokers:** Multi-broker support (Kite, Angel, Paper mode)
**Deployment:** Docker, Windows EXE (`build_exe.bat`)
**Safety:** Paper mode default, auto-trading disabled unless explicitly enabled, risk controls (drawdown, circuit breakers, kill switches)

### What This Bot Does
The bot automatically:
- Scans NSE index options (NIFTY / BANKNIFTY / FINNIFTY) every 30 seconds
- Generates trade signals based on RSI, MACD, ADX, volume, breakout, VWAP
- Scores each signal (0-100) and assigns a tier (STRONG / MODERATE / WEAK)
- Places trades automatically in PAPER mode (no real money)
- Sends Telegram alerts for every signal and exit
- Tracks P&L, drawdown, win rate in SQLite databases

### Entry Points

| Command | Purpose |
|---|---|
| `run_low_capital.bat` | Rs.5,000 PAPER mode launcher (recommended start) |
| `build_exe.bat` | Build Windows .exe launcher |
| `python index_app/index_trader.py --paper` | Manual PAPER mode start |
| `docker compose up -d` | Docker deployment (Linux/VPS) |

## 3. Workflow – How Trades Execute

```
Market Data Feed → NSE index options data ingested (1m/5m/15m bars)

IV Rank Check → VIX level checked (VIX > 27 = block all entries)

Session Check → Time of day evaluated
  09:20–10:15 IST = OPENING (-10 score penalty)
  10:15–11:30 IST = TRENDING (+5 score bonus) ← BEST WINDOW
  11:30–13:30 IST = CHOPPY (-15 score penalty) ← AVOID
  13:30–14:15 IST = RECOVERY (neutral)
  14:15–15:00 IST = PRE_CLOSE (-5 penalty)

Signal Engine → Pure indicators evaluated:
  • RSI: 40–60 bullish zone
  • MACD: Histogram positive = momentum
  • Breakout: Price above VWAP + volume > 1.2x
  • ADX: > 16 = trend confirmed
  • Volume ratio: > 1.2x average

Signal Score Computed (0–100) → Tier Assigned:
  Score 80+  → STRONG  tier (1 lot, 100% size)
  Score 70-79→ MODERATE tier (0.6 lot, 60% size)
  Score 60-69→ WEAK tier (0.3 lot, 30% size) — disabled by default
  Score <60  → IGNORE (no trade)

Risk Engine → Exposure limits checked:
  Daily loss < Rs.300? ✅
  Capital available ≥ lot cost? ✅
  Not in cooldown? ✅

Execution Engine → Order placed:
  PAPER mode = simulated fill (no real order)
  LIVE mode = real broker order

Trade Record → SQLite DB updated
Telegram Alert → Signal details sent to your phone

Exit Priority:
  1. Stop Loss    → Exit if option drops 8%
  2. Target Profit→ Exit if option rises 30%
  3. Trailing SL  → After +12% gain, trail at -5% from peak
  4. Time Exit    → Exit after 40 minutes max
```

## 5. Capital Settings for Rs.5,000

Edit `config.lowcap.json`:

```json
{
    "BASE_CAPITAL": 5000,
    "MAX_DAILY_LOSS": -300,
    "MAX_DRAWDOWN": 0.15,
    "MAX_TRADES_DAY": 1,
    "MAX_LOT_CAPITAL_PCT": 0.50,
    "AI_THRESHOLD": 60,
    "EXECUTION_MODE": "PAPER",
    "INDEX_PRIORITY": ["NIFTY", "BANKNIFTY"]
}
```

| Setting | Value | Meaning |
|---|---|---|
| BASE_CAPITAL | Rs.5,000 | Starting capital |
| MAX_DAILY_LOSS | -Rs.300 | Hard stop: bot stops if you lose Rs.300 in a day |
| MAX_DRAWDOWN | 15% | Bot stops if capital drops 15% from peak |
| MAX_TRADES_DAY | 1 | Only 1 trade per day (safe for small capital) |
| MAX_LOT_CAPITAL_PCT | 50% | Max Rs.2,500 per trade (50% of capital) |
| AI_THRESHOLD | 60 | Signal score must be 60+ to trade |

## 6. Safety Controls (Never Disable)

| Safety Feature | What It Does |
|---|---|
| **Paper Mode invariant** | PAPER mode never reaches real broker API |
| **_HARD_HALT** | Kills all entries when capital breach detected |
| **MAX_DAILY_LOSS** | Stops trading after Rs.300 daily loss |
| **MAX_DRAWDOWN** | Stops trading at 15% drawdown from peak |
| **Circuit Breaker** | Blocks entries if NSE/YF failure rate > 50% |
| **Expiry Cutoff** | No new entries after 13:30 IST on expiry day |
| **Session Blocks** | No entries after 15:00 IST daily |
| **Kill File** | Drop `STOP_TRADING` in project root → immediate halt |
| **VIX Block** | No entries when VIX > 27 |
| **Capital Lock** | Prevents double-spend on concurrent entries |

## 11. Final Notes

1. **Rotate secrets immediately** — edit `.env` with real Telegram credentials before LIVE mode
2. **Always start with PAPER mode** — run for at least 2 weeks to validate signals
3. **Rs.5,000 is the minimum** for NIFTY/BANKNIFTY 1-lot trading
4. **Never go LIVE without rotating secrets** — `.env` with live credentials + LIVE mode = real orders
5. **Check health weekly** — run `python -m core.health_checker` every Sunday
6. **Respect the hard stop** — if `MAX_DAILY_LOSS = -Rs.300`, stop the bot and review
7. **Scale up gradually** — after 2 weeks consistent profit, increase `BASE_CAPITAL`
