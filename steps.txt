Deployment Guide — Step by Step
Prerequisites

Python 3.10–3.19 (3.11 recommended)
pip install requests yfinance pandas numpy jsonschema
# Optional for ML:
pip install lightgbm scikit-learn
# Optional for PDF reports:
pip install reportlab
# Optional for news sentinel:
pip install feedparser scipy
Step 1 — Minimal config setup
Copy index_config.defaults.json to config.json and set only these keys:


{
  "BOT_TOKEN": "YOUR_TELEGRAM_BOT_TOKEN",
  "CHAT_ID": "YOUR_TELEGRAM_CHAT_ID",
  "BASE_CAPITAL": 50000,
  "MAX_DAILY_LOSS": -2500,
  "EXECUTION_MODE": "MANUAL",
  "BROKER_API_ENABLED": false,
  "manual_signal_enabled": true,
  "manual_signal_workflow_mode": "SIGNALS_ONLY",
  "telegram_commander_enabled": false
}
Everything else uses safe defaults from index_config.defaults.json.

Step 2 — Run in paper mode (recommended first)

# Terminal — paper simulation, no real orders ever
python index_app/index_trader.py --paper

# Or via GUI launcher
python launcher.py
The bot will:

Generate signals for NIFTY/BANKNIFTY/FINNIFTY
Send rich Telegram alerts (score bar, VIX/IV rank/PCR/regime/ADX/ML context)
Never touch any broker — pure signal mode
Step 3 — Enable manual signal queue (when ready)
In config.json:


{
  "manual_signal_workflow_mode": "FULL_MANUAL",
  "telegram_commander_enabled": true,
  "telegram_allow_live_position_cmds": false
}
You can now submit signals via Telegram:


/signal BANKNIFTY CALL 82 gap fill breakout
/pending                     ← see what's queued
/approve MSQ_1735xxx_0001    ← approve a specific signal
/reject  MSQ_1735xxx_0001    ← reject it
/approve_all                 ← approve everything pending
/status                      ← bot health + P&L
Or import from CSV:


python -m core.signal_importer --file signals.csv
Step 4 — Enable broker (when fully tested)
Only when you have 30+ profitable paper days:


{
  "EXECUTION_MODE": "AUTO",
  "BROKER_API_ENABLED": true,
  "BROKER_NAME": "kite",
  "manual_signal_workflow_mode": "MANUAL_PRIORITY",
  "telegram_allow_live_position_cmds": true,
  "BROKER_CONFIG": {
    "api_key": "YOUR_KEY",
    "access_token": "YOUR_TOKEN"
  }
}
No code changes needed — config toggle activates the live path.

Step 5 — Docker (production)

# Create .env file with secrets
echo "OPBUYING_BOT_TOKEN=..." >> .env
echo "OPBUYING_CHAT_ID=..."   >> .env
echo "OPBUYING_BASE_CAPITAL=50000" >> .env

# Start (paper mode by default)
docker compose up -d
docker compose logs -f opb

# Stop gracefully
docker compose down
Step 6 — Web dashboard (optional)

{ "web_dashboard_enabled": true, "web_dashboard_port": 8765 }
Open http://localhost:8765 — shows live signals, pending manual signals, approve/reject buttons, P&L, autopsy.

Build EXE (Windows standalone)
No .spec file exists yet. To create one:


pip install pyinstaller
pyinstaller --onefile --windowed \
  --add-data "index_config.defaults.json;." \
  --add-data "core;core" \
  --add-data "index_app;index_app" \
  launcher.py
# Output: dist/launcher.exe
The launcher.py already has frozen-binary detection code (sys.frozen) so it handles the PyInstaller case automatically.

Daily workflow summary

09:00  bot auto-starts (or restart manually)
       → loads pending signals from prior session
       → Telegram: "X signals pending from prior session"

09:15  market opens — scan loop starts
       → every 60s: scans NIFTY/BANKNIFTY/FINNIFTY
       → strong signals → rich Telegram message with GO/NO-GO context
       → you decide: /approve or /reject

15:20  market closes → EOD report sent
       → Telegram: trades, win rate, P&L, VaR, manual signal stats

Sunday → weekly health check + autopsy
